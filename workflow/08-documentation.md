# Stage 8 — Documentation & Memory Update

## Purpose

Record what was built, what was learned, and update project memory so future work benefits from this project's outcomes. This stage produces the definitive record of the project's results.

---

## Required Inputs

- Completed `07-review.md` (Stage 7)
- All previous project artifacts
- Any decisions made during the project
- Any reusable patterns discovered

---

## Key Questions to Answer

1. **What was built?** — Summary of what was delivered
2. **What key decisions were made?** — Are they all in `memory/decisions.md`?
3. **What patterns emerged?** — Anything reusable for future projects?
4. **What worked well?** — Process, tools, approaches that should be repeated
5. **What didn't work?** — What should be avoided or done differently next time
6. **What's the current state?** — Is it deployed? Where does it live?
7. **Are there known limitations?** — What doesn't it do that it should?
8. **What follow-up work exists?** — Is there a next phase?

---

## Process

1. Create `08-report.md` from `templates/post-task-report.md`
2. Review all decisions made — ensure `memory/decisions.md` is complete
3. Review patterns discovered — add new entries to `memory/patterns.md`
4. Update `memory/project-index.md` with the project summary
5. Update STATUS.md

---

## Output / Artifact

**File**: `08-report.md`
**Template**: [`templates/post-task-report.md`](../templates/post-task-report.md)

**Also update**:
- `memory/decisions.md` — ensure all decisions are recorded
- `memory/patterns.md` — add any new reusable patterns
- `memory/project-index.md` — add the project entry

---

## Exit Criteria

- [ ] `08-report.md` is complete with a clear summary of outcomes
- [ ] All decisions are recorded in `memory/decisions.md`
- [ ] Any reusable patterns are added to `memory/patterns.md`
- [ ] `memory/project-index.md` is updated with this project
- [ ] STATUS.md is updated to reflect completion of Stage 8
